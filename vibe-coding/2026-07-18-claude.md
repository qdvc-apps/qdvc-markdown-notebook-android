Done. Here's what changed.
The new workspace overview layer sits between the workspace list and the folder browser. Tapping a workspace now shows its name in the toolbar, the underlying folder path in a small monospace label, and three rows: Browse files (the folder structure, exactly as before), All notes (a recursive scan listing every markdown file in the workspace with the folder path it lives under), and Search (its own screen). Navigating into any of the three and pressing the OS back button returns to this overview; from the overview, back returns to the workspace list.
Search is a full-text search over note titles (file names and markdown headings) and body contents, case-insensitively. You type a query, tap Search, and each hit shows the note name, its folder path, and a ~120-character snippet around the first match.
The two thematic changes are in too: workspaces in the list now use a star icon (folders inside a workspace keep their folder icons), and re-tapping the Browse tab while already inside the Browse tab jumps straight back to the workspace list.
Under the hood, the browse state gained a BrowseMode (WORKSPACES/OVERVIEW/FOLDERS/ALL_NOTES/SEARCH) plus the active workspace and the all-notes/search data; browseUp() now walks that hierarchy and reports whether it consumed the back press. The recursive scan and search live in NoteRepository.
