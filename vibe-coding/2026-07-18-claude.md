## 1. User request

Please produce an Android app that can open a folder full of markdown notes and view/edit them.

The app has a bottom navigation bar with four icons (de facto tabs): Browse, View, Edit, Jump. When the app first starts up, only Browse and Jump are enabled; View and Edit are disabled.

### Browse

- The app starts in the Browse tab.
- Home page is a list of "workspaces" (each workspace is a folder full of markdown notes, i.e., files of extension `.md` or `.markdown`). User can add a new workspace, or remove an existing workspace.
- Tapping on a workspace shows the subfolders and markdown notes in the root of that workspace folder. Please show an animation as screen moves from the home page to this view. This view has a toolbar with a button for creating a new markdown note.
- Tapping on a markdown note results in it becoming the currently-open note. The screen jumps to the View tab. Because there is now a currently-open note, the View and Edit icons in the bottom navigation bar are now enabled.
- There should be a toolbar at the top, with a menu button on the right-hand side which pulls open a menu. At the moment there is only one item in this menu, Settings.
- The Settings screen is arranged as a series of menus and submenus (like a typical Android/iOS settings app).
- In the settings screen, so far there are two submenus:
    - 1. To switch between Automatic, Light Mode, Dark Mode. "Automatic" tries to pick up whatever the system option is.
    - 2. To configure Dark Mode style. One option is regular dark mode, the other is pure black mode.
    Whereas Regular Dark Mode might use dark shades of grey for certain UI elements/backgrounds, Pure Black Mode uses pure black for larger UI elements/backgrounds, to optimise for OLED displays. 

### View

- Displays the contents of the currently-open Markdown note (read-only).
- All text should be monospace font, all of the same font size. Markdown formatting is shown with syntax highlighting rather than varying the font size.

### Edit

- Same as the View mode, except it is no longer read-only; the user can make edits.
- There should be a toolbar at the top, with a button for saving the note if and only if changes have been made.

### Jump

- This is a phone app equivalent to the desktop app concept of a tab bar of various files that you have open, for quickly jumping between them.
- The currently-open note is listed, of course.
- Swiping a note listed here reveals a red "close" button (similar to the typical smartphone email app's "swipe to archive" gesture). Attempting to close a note with unsaved changes results in a warning that there are unsaved changes. The user can either "Cancel" (note does not get closed) or "Close anyway" (indicated in red - note is closed and changes are discarded).
- To open another note so that it is also listed here, the user simply has to tap on the Browse tab and navigate to another note and tap on it. Now that note becomes the currently-opened note and it is listed in the Jump tab as well.
- There should be a toolbar at the top with a button for rearranging the notes listed in the jump tab.
